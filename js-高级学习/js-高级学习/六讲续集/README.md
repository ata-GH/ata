## 设计模式
    javascript里面给我们提供了多种设计模式：工厂，桥接模式，组合模式，门面模式，
    适配器，享元，代理，观察者，命令，责任链等设计模式


### 工厂模式
    1.工厂模式，其核心就是为了生产对象，实现解耦。
    2.对于javascript而言，一个类或对象中往往会包含别的对象。
      我们创建这样的成员对象时，可能习惯采用new关键字或构造函数的方式，这样会导致两个类之间产生依赖关系。
      所以我们通常要需要工厂模式，创建一个工厂来管理它们的实例过程，这样可以控制程序，是我们的程序更加面向对象，多态化。
    3.实现简单工厂
    4.js中实现抽象类的概念，实现复杂的动态工厂。  


### 桥接模式
    1.桥接模式是一种既能把把两个对象连在一起，又能避免二者的强耦合的方法。
      通过‘桥’把彼此联系起来。同时又允许他们各自独立的变化。
    2.主要作用表现为：将抽象与其实现隔离开来，以便二者独立化  
    3.桥接实例：
        1.事件监听回调机制
        2.特权函数
        3.桥接组织多个单元
